# Contributing

Thank you for taking the time to contribute!

## Getting Started

1. Fork the repository and clone it locally.
2. Install dependencies: `npm install` (or the equivalent for your stack).
3. Create a branch: `git checkout -b feat/my-change`.

## Development

- Follow the code style enforced by the project's linter and formatter.
- Write tests for new behavior.
- Keep commits small and focused.
- Use [Conventional Commits](https://www.conventionalcommits.org/) for commit messages:
  - `feat:` — new feature
  - `fix:` — bug fix
  - `docs:` — documentation only
  - `chore:` — tooling, deps, build
  - `refactor:` — no behavior change
  - `test:` — tests only

## Submitting a Pull Request

1. Push your branch and open a PR against `main`.
2. Fill in the PR description: what changed and why.
3. Ensure all checks (CI, lint, tests) are green.
4. A maintainer will review and merge.

## Reporting Issues

Please use the issue tracker with a clear title and reproduction steps.
