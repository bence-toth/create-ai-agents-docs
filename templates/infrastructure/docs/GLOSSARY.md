# Glossary

Infrastructure-specific terms used throughout this codebase.

<!-- Add terms in alphabetical order. -->

| Term           | Definition                                                                                          |
| -------------- | --------------------------------------------------------------------------------------------------- |
| blast radius   | The scope of impact if a change goes wrong — how many resources or users would be affected         |
| drift          | A difference between the desired state in code and the actual state of deployed infrastructure      |
| idempotent     | An operation that produces the same result regardless of how many times it is applied               |
| module         | A reusable, parameterized unit of infrastructure code                                               |
| plan           | A preview of what changes will be made before they are applied                                      |
| state          | The record of what infrastructure currently exists, managed by the IaC tool                        |
