# Glossary

Domain-specific terms used throughout this codebase.

<!-- Add terms in alphabetical order. -->

| Term           | Definition                                                                                                |
| -------------- | --------------------------------------------------------------------------------------------------------- |
| API contract   | The agreed-upon shape and behavior of the API that both client and server must honour                     |
| client         | The frontend application running in the browser                                                           |
| server         | The backend API application running on the server                                                         |
| shared         | Code (types, validation schemas, constants) that is imported by both the client and the server            |
